InRule Rule Services Start Page

 
Welcome to InRule Rule Services!
InRule Rule Services is a reference architecture for providing WCF REST services for rule execution. This solution contains the following projects:
  • RuleServices - Rule Execution Services operations include: ApplyRules (Get), ExecuteRuleSet (Get), ExecuteRuleSet (Post)
  • HTMLTestUI - Simple html test page. Construct Rule Service request xml and execute request from JavaScript.
  • ASPMVCTestUI - Demonstrates calling Rule Services via ASP MVC using a Service Reference.
 
Testing Rule Services in Visual Studio 2012
Save test rule application to irCatalog:
  1. Perform a Complete local install of InRule including irCatalog (see InRule Installation Guide)
  2. Launch irAuthor (Start --> All Programs --> InRule --> irAuthor)
  3. Open sample rule application (Download Location/Rule Service/RuleServices45/InRule.RuleServices.SampleRuleApps/MortgageCalculator.ruleapp)
  4. Save rule application to irCatalog (File --> Save As --> Save to Catalog --> Click 'OK')

Steps for testing:

  1. Launch Visual Studio 2012
  2. Open RuleServices Solution (Download Location/Rule Service/RuleServices45/RulesServices.sln)
  3. Run the RuleServices solution - (InRule.RuleServices.HtmlTestUI should be set as StartUp Project)
    1. The following projects will start: RuleServcies, HTMLTestUI, ASPMVCTestUI
  4. StartPage (default.html) will launch in browser
  5. Test Rule Services by clicking on either the HTMLTestUI or ASPMVCTestUI
    1. Optional: Test Rule Services from another client (direct in browser, fiddler, etc.)
  6. Debug - set project to debug as StartUp Project or Attach to Process
Notes:
  • InRule.RuleServices.csproj is configured to start at http://localhost:50001/InRule.RuleServices/RuleExecutionService.svc/
  • InRule.HtmlTestUI.csproj is configured to start at http://localhost:50003/
  • InRule.ASPMVCTestUI.csproj is configured to start at http://localhost:50002/
  • For SOAP - uncomment SOAP behavior section Web.config, minor service interface and caller changes
 
Deploying Rule Services to IIS
Steps for deploying:
  1. XCopy the InRule.RuleServices dll and supporting InRule dlls (found in InRule.RuleServices\bin) to the designated virtual web directory.
  2. Change base address in services web.config and CatalogUri to relevant catalog service
